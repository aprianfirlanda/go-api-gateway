# Tenant Credentials

## Prerequisites

- Role is `tenant_admin`.
- Tenant admin API key configured when required (`VITE_TENANT_ADMIN_API_KEY`).

## Steps

1. Open `Tenant > Credentials`.
2. Create consumer (`Name`, `Slug`).
3. Issue API key credential.
4. Rotate credential using **Rotate** action.

## Expected Result

- Consumer and credential creation succeed for own tenant.
- New credential key prefix appears.
- Rotated key response returns newly issued key.

## Troubleshooting

- `401`: invalid tenant-admin key.
- `403`: tenant mismatch or insufficient role.
